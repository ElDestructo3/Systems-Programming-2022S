Question text

You use gprof to get the call graph of the following C program.

void f1(), f2(), f3();

void f1() { f2(); f3(); }
void f2() { f3(); }
void f3() { }

int main()
{
   int x, y, z, i;

   scanf("%d%d%d", &x, &y, &z);
   for (i=0; i<x; ++i) { f1(); f2(); }
   for (i=0; i<y; ++i) { f1(); f3(); }
   for (i=0; i<z; ++i) { f2(); f3(); }

   exit(0);
}

A part (contiguous) of the output supplied by gprof is given below.

-------------------------------------------------
             0.00   0.00     7/19        main [9]
             0.00   0.00    12/19        f1 [3]
[2]   0.0    0.00   0.00    19       f2 [2]
             0.00   0.00    19/40        f3 [1]
-------------------------------------------------

What are the values of x, y and z supplied by the user? Show the equations involving x,y,z that can be derived from this gprof output, with one sentence stating how each of these equations is obtained. Finally, write the solution for x,y,z. No need to show the detailed calculations.

The gprof output indicates that f2 has been called a total of 19 times , as from the third column. We note that f2 is called for a total of 2x+y+z times ( x from f1 in first loop, x from f2 in first loop, y from f1 in second loop and z from f2 in third loop).

Hence, 2x+y+z=19.

We also note from the first line of the output that f2 is called from main for a total of 7 times. Also main calls f2 directly x+z times (x in first loop, z in third loop).

Hence, x+z=7.

Finally, we note from the last line of the output thta f3 is called for a total of 40 times. Also, f3 is called for 3x+3y+2z times ( 2x from f1 and x from f2 in first loop, 2y from f1 and y from f3 in second loop, and z from f2 and z from f3 in third loop).

Hence, 3x+3y+2z=40

Solving, we get, x=5, y=7, z=2.

Question text

You have a C file splprog.c in which the strings (if any) do not contain the characters { and }. These characters are used solely to indicate the beginnings and the ends of blocks. Show how you can run grep in order to print only those lines in splprog.c, in which a block is both opened by { and closed by } in the same line. Only individual blocks in each line should match. For example, the following line should match like

   } else if (x > 0) { --i; ++j; } else { ++i; --j; }
                     -------------      -------------
but not like

   } else if (x > 0) { --i; ++j; } else { ++i; --j; }
                     --------------------------------

Assume that there are no nested blocks in the same line, that is, situations like { { } or { } } or { { } { } } do not occur in a line. Write only the grep command.

grep '{[^}]*}' splprog.c will give the expected output. 

Using '{.*}' will give the entire line (as in the second example) as output. In order to prevent that from happening, we use [^}]* to ensure that string matching force stops before the earliest closing brace, after which we can cover the closing brace.

Question text

Professor Foojit has a text file foonums.txt in which each line contains five positive integers separated by single spaces. There is no extra space at the beginning or at the end of any line. The integers may be arbitrarily large, but do not contain leading zero digits. Professor Foojit considers all integers in the range 2992999 unlucky. Out of these unlucky numbers, the number 593 is considered super-unlucky.

(a) [6 marks]
Write a grep command for Professor Foojit to print all the lines in foonums.txt, that do not contain any unlucky number. Write only the grep command.

(b) [8 marks]
Write a single sed command for Professor Foojit that modifies and prints the lines in foonums.txt as follows. If the line does not contain the super-unlucky number 593, then the line is printed as it is. If the line contains one or more occurrences of the super-unlucky number, then only the first occurrence of the number in that line is replaced by the text BADNUM. For example, consider the following lines.

589 590 591 592 593
593593 3593 5934 59 5
593 594 595 596 597
59 593 5937 593 593

These lines are to be printed as follows.

589 590 591 592 BADNUM
593593 3593 5934 59 5
BADNUM 594 595 596 597
59 BADNUM 5937 593 593

(a) grep -vw '[3-9][0-9][0-9]\|[1-2][0-9][0-9][0-9]\|299' foonums.txt

(b) sed  's/\b593\b/BADNUM/' foonums.txt
